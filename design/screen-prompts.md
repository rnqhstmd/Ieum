## A. design.md(무드) 잘 넣는 법

1. **Design System으로 등록 / 첨부**: Claude Design 상단 "Design system" 드롭다운에 `spacex-DESIGN.md`를 시스템으로 넣으면 모든 화면에 일관 적용됩니다. (불가하면 매 프롬프트 맨 위에 design.md 전문을 붙여넣기)
2. **단일 기준(SSOT) 선언**: 프롬프트 맨 앞에 → *"첨부한 design.md를 색·타이포·간격·컴포넌트의 단일 기준으로 삼아라. 토큰과 Do/Don't를 지켜라."*
3. **적응 규칙 명시(아래 B)**: design.md는 마케팅용이라, "이건 생산성 앱"이라는 적응 규칙을 반드시 같이 줘야 한다.
4. **한 번에 한 화면**: 화면 하나씩 생성하고, 다음 화면은 *"앞 화면들과 동일 시스템/컴포넌트로 일관되게"* 라고 이어준다.
5. **토큰으로 말하기**: "near-black `#0a0a0a` 서피스", "ghost pill 버튼", "헤어라인 `#3a3a3f`"처럼 design.md 토큰을 직접 인용.

## B. SpaceX 무드 → 생산성 앱 적응 규칙 (모든 화면 공통)

- **유지**: 검정/흰색 베이스, D-DIN(없으면 Inter 700·uppercase·넓은 트래킹)으로 **nav·섹션 라벨·버튼·eyebrow**, 고스트 pill 버튼, 헤어라인 구분선, 그림자 없는 평면, "엔지니어드한" 정밀함.
- **적응(필수)**:
  1. **본문·문서·에디터 텍스트는 sentence-case + line-height 1.5~1.7** (대문자 강제 금지 — 대문자는 chrome/헤딩/버튼에만).
  2. **기능용 presence 색 4~6개 허용** (사용자별 라이브 커서·아바타 구분용. 브랜드 액센트가 아니라 협업 기능).
  3. **작업 화면 서피스 = `#000000`/`#0a0a0a` + 헤어라인** (full-bleed 사진은 **랜딩에만**).
  4. **선택·호버·포커스 상태**는 헤어라인/미세 명도 차이로 표현(그림자·글로우 금지).
- **반응형**: 데스크탑 2-pane → 태블릿 사이드바 토글 → 모바일 사이드바 드로어·모달은 바텀시트/풀스크린. 디스플레이 80→60→48→40px 계단.
- **스택**: Next.js(App Router)+React+Tailwind로 구현 가능, 재사용 컴포넌트 단위.

## D. 화면별 프롬프트 (각각 복붙)

### 1. 랜딩 (비로그인) — 유일하게 full-bleed 히어로 허용
```
[공통 프리앰블] +
랜딩 페이지. full-bleed 다크 히어로(추상적인 '연결/노드/별자리' 비주얼 — 사진 대신 미니멀 그래픽 가능) 위에 대문자 D-DIN 헤드라인 1줄 + 짧은 서브카피 + 고스트 pill CTA "Google로 시작" 1개. 상단에 오버레이 nav(로고 'IEUM' 워드마크 + 로그인 링크). 스크롤하면 제품 핵심(실시간 공동편집·중첩 페이지·presence)을 흑백 밴드 2~3개로. 푸터(dark). 모바일은 nav 햄버거, 헤드라인 40px.
```

### 2. 로그인
```
[공통 프리앰블] +
로그인 화면. 검정 캔버스 중앙에 'IEUM' 워드마크 + 한 줄 카피 + 고스트 pill "Google로 로그인" 버튼 1개. 그 외 요소 최소. 하단에 약관/개인정보 caption. 모바일은 풀폭 중앙 정렬.
```

### 3. 메인 앱 화면 (사이드바 + 페이지 에디터 + presence) — ⭐ 핵심, 가장 공들이기
```
[공통 프리앰블] +
로그인 후 메인 앱 화면. 좌측 고정 사이드바 + 우측 페이지 에디터의 2-pane (다크 #0a0a0a + 헤어라인).
[사이드바] 상단 워크스페이스 스위처(개인 워크스페이스 + 공유 워크스페이스 목록), 그 아래 페이지 트리(중첩 들여쓰기·펼침/접힘 chevron·호버 시 '+'와 '...'), "새 페이지"/"새 워크스페이스" 버튼(고스트), 하단 계정 영역(아바타+이름→메뉴).
[에디터] 상단 페이지 아이콘(이모지)+제목(인라인 편집, sentence-case), 본문은 블록 에디터(단락/제목1~3/글머리 목록 — 본문은 읽기 좋은 sentence-case·행간 1.6), "저장됨" 미세 표시.
[presence] 우상단에 접속자 아바타 스택 + "3명 보는 중", 본문 안에 다른 사용자 라이브 커서 2개(서로 다른 기능 색 + 이름 라벨).
모바일(<768): 사이드바는 햄버거→드로어, 에디터 풀폭, presence 아바타 압축. 데스크탑/모바일 둘 다 제시.
```

### 4. 공유 워크스페이스 멤버 관리 + 초대 (모달)
```
[공통 프리앰블] +
공유 워크스페이스 멤버 관리 모달(데스크탑) / 바텀시트(모바일). 구성: 멤버 목록(아바타+이름+역할 뱃지 OWNER(관리자)/MEMBER + OWNER에게만 보이는 '역할변경'·'내보내기'), 상단에 이메일 초대 입력 + "초대" 고스트 버튼, 그 아래 '보류 중 초대' 목록(이메일+만료까지 남은 일수+취소). 권한 없는 사용자에겐 관리 액션 숨김. 헤어라인으로 행 구분, 그림자 없음.
```

### 5. 초대 수락 화면
```
[공통 프리앰블] +
초대 링크 진입 화면. 검정 캔버스 중앙 카드: "OOO 워크스페이스에 초대되었습니다" + 초대한 사람/역할 표시 + 고스트 pill "수락" 버튼. 상태 변형 3개도 같이: (a) 만료된 초대, (b) 이미 멤버, (c) 유효하지 않은 토큰. 모바일 풀폭.
```

### 6. 상태/엣지 + 페이지 컨텍스트 메뉴
```
[공통 프리앰블] +
다음 상태들을 한 보드에 모아서: (1) 빈 워크스페이스(페이지 없음) 빈 상태 + "첫 페이지 만들기" CTA, (2) 로딩 스켈레톤(사이드바+에디터), (3) 권한 없음(403) 화면, (4) 오프라인/재연결 배너, (5) 페이지 우클릭 컨텍스트 메뉴(이름변경·아이콘·하위 페이지 추가·아카이브). 전부 다크 서피스+헤어라인, 에러는 미세 색만.
```
