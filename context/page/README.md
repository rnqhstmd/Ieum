# 페이지·에디터 (page)

워크스페이스 내 문서를 무한 중첩 트리로 조직하고, 블록 기반 contenteditable 에디터로 작성·편집하는 핵심 도메인.

## 배경

Ieum은 Google 로그인 기반 노션 유사 실시간 협업 문서 서비스다. 페이지 도메인은 그 중심축으로, 사용자가 문서를 계층적으로 구성(중첩 페이지 트리)하고 내용을 작성(블록 에디터)할 수 있도록 한다. 에디터 콘텐츠는 자체 구현 RGA CRDT 상태로 관리되며, 단일 사용자 편집과 다중 사용자 협업 편집 모두 동일한 에디터 위에서 동작한다. 실시간 동기화 자체는 collaboration 도메인 소관이며, 이 문서는 페이지 구조와 단일 사용자 편집 모델에 집중한다. 신규 구축 프로젝트(Phase 1).

## 안 하면 어떻게 되는가

- 문서를 생성하고 편집할 수단이 없어 서비스 자체가 성립하지 않는다.
- 트리 구조가 없으면 문서 조직이 불가능해 단순 메모장 수준에 머문다.
- Soft delete 없이 즉시 삭제하면 하위 페이지까지 데이터가 영구 손실된다.
- 자동저장이 없으면 편집 중 탭 전환·새로고침 시 내용이 유실된다.

## 사용자와 규모

| 구분 | 내용 |
|------|------|
| 대상 사용자 | Google 계정 보유자 (개인·팀) |
| 워크스페이스 유형 | 개인(PERSONAL) / 공유(SHARED) |
| MVP 동시 접속 목표 | 워크스페이스당 최대 10명 |
| 페이지 트리 깊이 | 무한 중첩 허용 (MVP 실용 범위 내) |
| 개발 체계 | 1인 사이드 프로젝트 (rnqhstmd), 데스크톱 브라우저 우선 |

## 성공 기준

| 항목 | 기준 |
|------|------|
| 페이지 CRUD | 워크스페이스 내 페이지 생성·조회·제목/아이콘 변경·soft delete 동작 |
| 무한 중첩 | parentPageId 자기참조로 깊이 제한 없이 하위 페이지 생성 가능 |
| 트리 탐색 | 사이드바에서 부모 펼침 시 하위 페이지 목록 표시 |
| Soft delete | archivedAt 설정으로 삭제, 하위 페이지 재귀 아카이브, 트리에서 숨김 |
| position 정렬 | gap-based 정수 간격(1000 단위), 중간 삽입은 평균값, 간격 소진 시 rebalance |
| 블록 편집 | paragraph / heading1~3 / bullet list 타입, Enter 새 블록·Backspace 빈 블록 삭제 |
| 자동저장 | debounce 500ms 후 저장 (협업 시 CRDT op로 즉시 전송) |
| 에디터 일관성 | 단일 사용자·협업 모드 모두 동일 contenteditable 에디터 동작 |
| 복원 UI | ❓ MVP에서 아카이브 복원 UI는 선택적으로 구현 |
| 드래그앤드롭 정렬 | ❓ position 필드는 유지하나 UI는 post-MVP로 연기 가능 |
| 이모지 피커 UI | ❓ 직접 입력 허용, 피커 UI는 post-MVP |
