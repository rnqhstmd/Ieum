# 인증·권한 (auth)

Google OAuth 기반 로그인·세션 관리와 워크스페이스·페이지 리소스에 대한 역할별 접근 제어를 담당하는 도메인.

## 배경

Ieum은 실시간 협업 문서 서비스로, 다수의 사용자가 워크스페이스와 페이지를 공유한다.
신규 구축 단계(Phase 0)이며 별도 계정 시스템 없이 Google OAuth를 단일 인증 수단으로 채택했다.
Auth.js(NextAuth v5)를 통해 OAuth 흐름을 처리하고, JWT 세션 전략으로 서버 상태 없이 인증을 유지한다.

## 안 하면 어떻게 되는가

- 인증 없음: 누구나 타인의 워크스페이스·페이지에 무단 접근 가능
- 세션 미관리: 브라우저 탭 이동·새로고침마다 재로그인 필요, 실시간 서버(WebSocket)가 요청자를 식별 불가
- 권한 모델 없음: MEMBER가 다른 멤버를 제거하거나 워크스페이스를 삭제하는 권한 상승 가능
- WebSocket 인가 없음: 악의적 클라이언트가 타인의 userId를 siteId에 위조해 CRDT op 전송 가능

## 사용자와 규모

| 구분 | 내용 |
|------|------|
| 대상 | Google 계정 보유자 (일반 개인 사용자) |
| 1인 개발 | 개발자 = rnqhstmd (사이드 프로젝트) |
| 동시 접속 목표 | 워크스페이스당 10명 (MVP 목표, 출처: 02-prd.md 비기능·성능) |
| CRDT op 브로드캐스트 지연 | p95 < 300ms (로컬 환경) |
| 세션 만료 | JWT maxAge 30일, Refresh Token 없음 (재로그인으로 갱신) |

## 성공 기준

| 기준 | 측정 방법 |
|------|-----------|
| Google OAuth 버튼 클릭 → Google 계정 선택 화면으로 이동 | 수동·E2E 검증 |
| 최초 로그인 시 User 레코드 + 개인 워크스페이스 자동 생성 | DB 상태 확인 |
| 재로그인 시 기존 User 레코드 재사용 (googleId 기준 upsert) | DB 상태 확인 |
| 로그아웃 후 보호 경로 접근 → `/login` 리다이렉트 | Playwright E2E |
| 만료/무효 세션으로 API 호출 → 401 반환 | API 통합 테스트 |
| 로그인 성공 후 callbackUrl 페이지로 복귀 | Playwright E2E |
| MEMBER가 OWNER 전용 액션 시도 → 403 반환 | API 통합 테스트 |
| 미인증 WebSocket 연결 시도 → ws.close(4001) | WebSocket 통합 테스트 |
| 워크스페이스 비멤버의 페이지 접근 → 403 반환 | API 통합 테스트 |
| 동시 초대 수락 경쟁 조건에서 중복 Membership 미생성 ❓ | 부하 테스트 (시나리오 미정) |

> ❓ = 정량 검증 방법이 현재 기획 문서에 명시되지 않음
