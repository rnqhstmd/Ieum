# 워크스페이스 (workspace)

사용자의 문서를 담는 컨테이너 단위로, 개인 전용(PERSONAL)과 팀 협업용(SHARED) 두 유형을 지원하며 멤버십·초대·역할 관리를 통해 접근을 제어한다.

## 배경

Ieum은 Google 로그인 기반의 노션 유사 실시간 협업 문서 서비스다. 사용자가 문서를 생성하고 편집하려면 먼저 워크스페이스가 필요하며, 워크스페이스는 다음 두 가지 방식으로 존재한다.

- **개인 워크스페이스 (PERSONAL)**: 최초 로그인 시 자동 생성되는 1인 전용 공간. 별도 설정 없이 즉시 문서 작성 가능.
- **공유 워크스페이스 (SHARED)**: 사용자가 직접 생성하고 이메일로 팀원을 초대해 함께 편집하는 협업 공간. OWNER/MEMBER 역할로 권한을 분리한다.

모든 페이지는 반드시 하나의 워크스페이스에 속하며, 워크스페이스 밖에 존재하는 페이지는 없다. 신규 구축 프로젝트이므로 기존 데이터 마이그레이션 고려 없이 설계 가능하다.

## 안 하면 어떻게 되는가

- 개인 워크스페이스 자동 생성이 없으면 사용자가 로그인 직후 빈 화면을 마주하고 문서 작성을 시작할 수 없다.
- 공유 워크스페이스·초대 흐름이 없으면 협업이 불가능하여 서비스 핵심 가치(실시간 공동편집)를 제공할 수 없다.
- OWNER/MEMBER 역할 구분이 없으면 임의의 멤버가 다른 멤버를 내보내거나 워크스페이스를 삭제하는 사고가 발생한다.
- 마지막 OWNER 보호 규칙이 없으면 워크스페이스가 관리자 없는 상태로 방치될 수 있다.

## 사용자와 규모

| 구분 | 내용 |
|------|------|
| 1차 사용자 | 개인 사용자 (개인 워크스페이스) |
| 2차 사용자 | 소규모 팀 (공유 워크스페이스) |
| MVP 동시 접속 목표 | 워크스페이스당 10명 |
| 워크스페이스 이름 제한 | 1자 이상 100자 이하 |
| 개인 워크스페이스 | 계정당 1개 고정 (추가 생성·삭제 불가) |
| 초대 토큰 유효 기간 | 7일 (expiresAt) |

> 동시 접속 10명 기준은 부하 테스트(post-MVP)로 검증 예정이다.

## 성공 기준

| 기준 | 상태 |
|------|------|
| 최초 로그인 시 PERSONAL 워크스페이스 1개 자동 생성 | ⬜ |
| 개인 워크스페이스 1개 제한 (애플리케이션 레이어 강제) | ⬜ |
| 공유 워크스페이스 생성 시 생성자에게 OWNER Membership 자동 부여 | ⬜ |
| 사이드바에 개인·공유 워크스페이스 목록 표시 및 전환 | ⬜ |
| OWNER만 공유 워크스페이스 삭제·멤버 초대·멤버 제거·역할 변경 가능 | ⬜ |
| 초대 링크(7일 유효) 클릭 시 Membership 자동 생성 | ⬜ |
| 초대 상태 PENDING → ACCEPTED \| REVOKED \| EXPIRED 전이 동작 | ⬜ |
| 마지막 OWNER는 역할 변경·나가기 불가 (경고 표시) | ⬜ |
| 비멤버의 워크스페이스 페이지 접근 시 403 반환 | ⬜ |
| 워크스페이스 삭제 시 하위 페이지·멤버십 cascade 삭제 | ⬜ |

❓ **불명확 사항**

- 공유 워크스페이스 이름 초기값 규칙 (개인 워크스페이스는 `{name}의 워크스페이스`로 명시됨, 공유는 미명시)

**확정된 기본값**

- **초대 이메일 발송**: MVP부터 Resend를 통해 실제 이메일 발송. "TODO: 이메일 전송" 플레이스홀더 없음.
- **Invitation 만료 스케줄러**: Vercel Cron 일 1회 일괄 처리(`PENDING → EXPIRED`) + 수락 시 lazy 검사 병행. 배포 환경: Vercel.
- **워크스페이스 멤버 수 상한**: 하드 상한 없음. 동시 접속 10명은 성능 목표(부하 테스트 post-MVP)이며 멤버십 수 제한과 별개.
