# PRD: A3 인프라 컴포넌트 3종 라이브 배선 (커맨드 팔레트 · 오류 토스트 · 연결 상태 배너)

## 배경
CommandPalette(⌘K 검색), ErrorToast(오류 토스트), ConnectionBanner(연결 상태 배너)는 이미 만들어져 있으나 #38 쇼케이스에만 있고 실제 제품 화면에 연결되지 않았다. 사이드바 "검색 ⌘K" 박스는 시각 전용 placeholder(비기능)다. 제목 저장 실패는 현재 유일하게 사용자 알림이 전무한 지점(조용히 idle 복귀)이다. 공동 편집 연결 끊김/복구를 알리는 요소가 없다(실시간 연결은 페이지 편집 화면에서만 성립). 직전 슬라이스(#39/#40)에서 6개 컴포넌트를 배선했고 이 3개는 "인프라 필요"로 명시적으로 미뤄졌다.

## 목표
⌘K/검색창으로 페이지 즉시 이동. 저장 실패 등 오류를 명확히 알리고 재시도 제공. 연결 끊김/복구 상태 인지.

## 요구사항

### 커맨드 팔레트
- [Must] FR-1: 어디서든 ⌘K(Mac)/Ctrl+K(Win/Linux)로 팔레트가 열린다.
- [Must] FR-2: 사이드바 "검색" 박스 클릭으로도 팔레트가 열린다(placeholder→실트리거).
- [Must] FR-3: 검색창 입력 시 현재 워크스페이스 페이지 중 제목 일치 항목만 실시간 필터.
- [Must] FR-4: 결과 항목 클릭/선택 시 해당 페이지로 이동 + 팔레트 닫힘.
- [Must] FR-5: Escape/바깥 클릭 시 닫힘(기존 유지).
- [Should] FR-6: 검색어 비었을 때 전체 페이지 목록을 그룹으로 표시(빈 화면 X).
- [Could] FR-7: 위/아래 방향키로 결과 하이라이트 이동.

### 오류 토스트
- [Must] FR-8: 화면 비종속 공용 토스트 알림 기능 마련.
- [Must] FR-9: 제목 저장 실패 시 오류 토스트 노출.
- [Must] FR-10: "다시 시도" 클릭 시 직전 실패 저장을 동일 내용으로 재호출.
- [Must] FR-11: 닫기(X)로 즉시 닫힘 + 조작 없으면 일정 시간 후 자동 소멸.
- [Should] FR-12: 새 오류는 기존 토스트를 최신 내용으로 교체(중복 누적 X).

### 연결 상태 배너
- [Must] FR-13: 편집 화면에서 연결 끊기면 상단 "오프라인" 배너.
- [Must] FR-14: 복구 시 "다시 연결됨" 배너 잠시 표시 후 자동 소멸.
- [Must] FR-15: 끊긴 적 없는 정상 상태에선 배너 미노출.

### 비즈니스 규칙
- [Must] BR-1: 팔레트 검색은 현재 워크스페이스로 한정.
- [Must] BR-2: 토스트 "다시 시도"는 방금 실패한 동일 작업만 재시도.
- [Should] BR-3: 연결 배너는 실시간 연결이 있는 편집 화면에서만 노출.

## 수용 기준
- AC-1: 어느 화면에서든 ⌘K/Ctrl+K → 팔레트 열림 [FR-1]
- AC-2: 사이드바 검색 박스 클릭 → 팔레트 열림 [FR-2]
- AC-3: 현재 워크스페이스 페이지 제목 일부 입력 → 해당 문자열 포함 페이지만 목록에 남음 [FR-3]
- AC-4: 결과 클릭 → 해당 페이지 이동 + 팔레트 닫힘 [FR-4]
- AC-5: Escape → 팔레트 닫힘 [FR-5]
- AC-6: 검색어 없이 열면 워크스페이스 페이지 목록 그룹 표시 [FR-6]
- AC-7: 아래/위 방향키로 하이라이트 이동 [FR-7]
- AC-8: 제목 저장 외 상황에서도 동일 방식으로 토스트 노출 가능(화면 비종속) [FR-8]
- AC-9: 제목 저장 API 실패 → "변경사항을 저장하지 못했습니다" 토스트 [FR-9]
- AC-10: "다시 시도" → 직전 실패 제목 저장을 동일 내용으로 재호출 [FR-10, BR-2]
- AC-11: 토스트 후 지정 시간(기본 5초) 경과 시 조작 없이 사라짐 [FR-11]
- AC-12: 닫기(X) → 자동 소멸 전이라도 즉시 사라짐 [FR-11]
- AC-13: 제목 저장 연속 2회 실패해도 토스트는 하나만(최신 내용 갱신) [FR-12]
- AC-14: 편집 화면 연결 끊김 → 상단 오프라인 배너 [FR-13]
- AC-15: 복구 → "다시 연결됨" 전환 후 자동 소멸 [FR-14]
- AC-16: 끊긴 적 없는 정상 진입 → 배너 미렌더 [FR-15]
- AC-17: 검색 결과에 현재 워크스페이스 페이지만(타 워크스페이스 X) [BR-1]
- AC-18: 대시보드·멤버 관리 등 연결 없는 화면엔 배너 미렌더 [BR-3]

## 제외 범위
- 백엔드 신규 검색 API(기본안: 클라이언트 로드 목록 검색)
- 팔레트 "명령 실행"(테마·로그아웃)·"사람 찾기"(기본안: 페이지 이동만)
- 모든 API 실패의 토스트 통합(기본안: 제목 저장 실패만)
- 오프라인 로컬 편집 큐잉(상태 표시와 별개)
- 컴포넌트 시각 디자인 변경(#38 확정 사용)

## 스코프 확정 (사용자 선택 — phase-requirements Q&A)
- Q1 검색 범위: **페이지 이동만**(현재 워크스페이스 페이지 제목 검색·이동). 사람/명령 검색은 후속.
- Q2 토스트 트리거: **제목 저장 실패만**(usePageTitle.saveTitle 실패). 멤버 alert·사이드바 인라인은 미변경.
- Q3 연결 배너 노출: **에디터(페이지) 화면 한정**. 대시보드·멤버 등 미노출.
- Q4 검색 API: **클라이언트 로드 목록으로 검색**(신규 백엔드 API 없음). → **백엔드 무변경, 프론트만**.
