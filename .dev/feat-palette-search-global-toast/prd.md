# PRD: CommandPalette(⌘K) 명령·사람 검색 확장 + ErrorToast 전역화

## 배경

**기능 A**: ⌘K 팔레트(CommandPaletteContainer + CommandPalette)는 현재 워크스페이스 페이지 트리를 "페이지" 그룹 한 개로만 검색·이동시켜준다. 검색창 placeholder는 이미 "페이지 이동, 사람 찾기, 명령 실행…"이며, 재사용 뷰(CommandPalette)는 여러 그룹과 그룹을 가로지르는 전역 하이라이트 인덱스를 이미 지원한다. 실제 명령 실행/멤버 검색 기능은 없다. 설정/도움말 이동·테마 전환·로그아웃은 계정 메뉴에서만, 멤버 관리 진입은 사이드바 "멤버" 버튼(공유 ws 전용)으로만 가능하다.

**기능 B**: 전역 토스트 인프라(ToastProvider/useToast)는 이미 (app) 레이아웃 전체를 감싸지만 사용처는 에디터 제목 저장 실패 한 곳뿐이다. MembersModal 변경 액션 실패 시 블로킹 `alert()`가 뜨고, Sidebar는 페이지 생성/이름변경/아이콘/아카이브 실패 시(401 제외) 초기 트리 조회 실패와 같은 `setStatus('error')` 경로를 타서 사이드바 트리 전체가 오류 화면으로 바뀐다(트리를 이미 불러왔는데도 이름 하나 바꾸다 실패하면 목록이 사라짐).

## 요구사항

### 기능 A — CommandPalette 확장

- [Must] FR-A1: "명령 실행" 그룹은 6개 중 현재 컨텍스트에서 유효한 것만 노출(검색어는 노출 후보에 대해 부분일치·대소문자 무시): ①새 페이지 만들기(선택 ws 있을 때) ②설정 열기(상시) ③도움말 열기(상시) ④테마 전환(상시) ⑤멤버 관리 열기(SHARED 확정 시) ⑥로그아웃(상시).
- [Must] FR-A2: "새 페이지 만들기" → 사이드바 "+ 새 페이지"와 동일 생성 흐름(현재 ws 최상위, 빈 제목, 형제 max position+1), 선택 즉시 닫힘, 성공 시 그 페이지로 이동.
- [Must] FR-A3: "설정 열기" → `/settings` 이동 + 닫힘.
- [Must] FR-A4: "도움말 열기" → `/help` 이동 + 닫힘.
- [Must] FR-A5: "테마 전환" → 다크↔라이트 반전(계정 메뉴 토글과 동일), 이동 없이 닫힘.
- [Must] FR-A6: "멤버 관리 열기" → `/workspace/{wsId}/members` 이동 + 닫힘. PERSONAL에서는 후보 자체에 없음.
- [Must] FR-A7: "로그아웃" → 로그아웃 실행. 성공 시 `/login` 이동, 실패 시 세션 유지·조용히 무시(계정 메뉴와 동일). 선택 즉시 닫힘(성공/실패 무관).
- [Must] FR-A8: "사람 찾기" 그룹 추가. SHARED일 때만, 멤버 이름/이메일 부분일치(대소문자 무시) 후보. PERSONAL에서는 그룹 자체 미렌더.
- [Must] FR-A9: 사람 항목 선택 → `/workspace/{wsId}/members` 이동 + 닫힘(전용 프로필 페이지 없음).
- [Must] FR-A10: 세 그룹(페이지·사람·명령)은 동일 검색어로 동시 필터. 항목 0개 그룹은 그룹 제목까지 렌더 제외.
- [Must] FR-A11: 방향키(↑/↓)는 보이는 전체 항목을 그룹 경계 무관하게 "페이지 → 사람 → 명령" 순 순회(경계 클램프). Enter는 하이라이트 항목 동작 실행.
- [Must] FR-A12: 사람 후보는 팔레트가 열릴 때(SHARED) 조회. 조회 완료 전엔 그룹 미노출. 401→`/login`. 비401 오류→"사람" 그룹만 계속 숨김, 나머지 정상.
- [Must] FR-A13: 사이드바 초기 트리 조회 중(loading)엔 팔레트 전체가 "불러오는 중…"만 표시(기존 회귀 방지).
- [Must] FR-A14: 닫았다 다시 열면 검색어 초기화 + 세 그룹 초기 재노출.
- [Must] BR-A1: 6개 명령·문구는 고정 목록(멤버 관리 조건부 노출 제외, 사용자별 상이 없음).
- [Should] QE-A1: 기존 "페이지" 그룹 검색·클릭·방향키 동작 회귀 없음.

### 기능 B — ErrorToast 전역화

- [Must] FR-B1: MembersModal 변경 액션(초대/역할/내보내기/초대취소) 실패 → `alert()` 대신 `useToast().showError`. 문구 "작업을 처리하지 못했습니다. 잠시 후 다시 시도해 주세요." 유지. 401→`/login`, 이때 토스트 없음.
- [Must] FR-B2: Sidebar 페이지 변경 액션(생성/이름변경/아이콘/아카이브확정) 비401 실패 → 트리 전체 오류화 금지. 전역 토스트로 알리고 트리는 직전 상태 유지. 401→`/login`, 토스트 없음.
- [Must] FR-B3: 초기 ws 목록 조회·최초/전환 트리 조회 실패는 대상 아님 — 기존 인라인 오류("목록을 불러오지 못했습니다…") 유지.
- [Must] FR-B4: 기능 A "새 페이지 만들기" 경유 생성 실패도 FR-B2/BR-B1 규칙 동일 적용.
- [Must] BR-B1: Sidebar mutation 실패 토스트 문구(고정): 생성→"페이지를 만들지 못했습니다. 잠시 후 다시 시도해 주세요." / 이름변경→"이름을 변경하지 못했습니다. 잠시 후 다시 시도해 주세요." / 아이콘→"아이콘을 변경하지 못했습니다. 잠시 후 다시 시도해 주세요." / 아카이브→"페이지를 아카이브하지 못했습니다. 잠시 후 다시 시도해 주세요."
- [Could] BR-B2: Sidebar mutation 실패 토스트에 "다시 시도" 버튼(에디터 저장 실패 패턴). 필수 아님.

## 수용 기준

### 기능 A

AC-A1: 개인 ws 기본 명령 노출
  Given: 현재 ws가 PERSONAL, 검색어 비어있음
  When: ⌘K로 팔레트를 연다
  Then: "명령 실행"에 "새 페이지 만들기/설정 열기/도움말 열기/테마 전환/로그아웃" 5개 렌더, "멤버 관리 열기" 텍스트는 문서에 없음. → [FR-A1/BR-A1]

AC-A2: 공유 ws 전체 명령 노출
  Given: 현재 ws가 SHARED
  When: 검색어 없이 팔레트를 연다
  Then: "명령 실행"에 "멤버 관리 열기" 포함 6개 모두 렌더. → [FR-A1/FR-A6]

AC-A3: 새 페이지 만들기 실행
  Given: 팔레트 열림, "새 페이지 만들기" 보임
  When: 해당 항목을 클릭한다
  Then: 생성 흐름(현재 ws 최상위, 빈 제목) 트리거 + 클릭 즉시 onClose 호출, 성공 시 onNavigate(생성 페이지). → [FR-A2]

AC-A4: 설정 열기
  Given: 팔레트 열림
  When: "설정 열기" 클릭
  Then: router.push('/settings') + 닫힘. → [FR-A3]

AC-A5: 도움말 열기
  Given: 팔레트 열림
  When: "도움말 열기" 클릭
  Then: router.push('/help') + 닫힘. → [FR-A4]

AC-A6: 테마 전환
  Given: 팔레트 열림, 현재 테마 다크
  When: "테마 전환" 클릭
  Then: 테마 토글 호출 → 라이트 전환, 라우터 이동 없음, 닫힘. → [FR-A5]

AC-A7: 멤버 관리 열기
  Given: SHARED, "멤버 관리 열기" 보임
  When: 클릭
  Then: router.push('/workspace/{wsId}/members') + 닫힘. → [FR-A6]

AC-A8: 로그아웃 성공
  Given: 팔레트 열림
  When: "로그아웃" 클릭 + 요청 성공
  Then: router.push('/login'), 팔레트는 클릭 시점 이미 닫힘. → [FR-A7]

AC-A9: 로그아웃 실패
  Given: 팔레트 열림
  When: "로그아웃" 클릭 + 요청 실패
  Then: 라우터 이동 없음(세션 유지), 팔레트는 클릭 시점 이미 닫힘. → [FR-A7]

AC-A10: 사람 찾기 이름 검색
  Given: SHARED, 멤버 "홍길동"(hong@ex.com)/"김철수"(kim@ex.com)
  When: "길동" 입력
  Then: "사람 찾기"에 "홍길동"만, "김철수" 문서에 없음. → [FR-A8]

AC-A11: 사람 찾기 이메일 검색
  Given: AC-A10 동일 멤버
  When: "kim@" 입력
  Then: "사람 찾기"에 "김철수"만. → [FR-A8]

AC-A12: 사람 항목 선택
  Given: "사람 찾기"에 "홍길동" 보임
  When: 클릭
  Then: router.push('/workspace/{wsId}/members') + 닫힘. → [FR-A9]

AC-A13: 개인 ws 사람 그룹 숨김
  Given: PERSONAL
  When: 팔레트를 연다
  Then: "사람 찾기" 그룹 제목 텍스트가 문서에 렌더되지 않음. → [FR-A8]

AC-A14: 빈 그룹 헤더 숨김
  Given: SHARED, 페이지≥1·멤버≥1
  When: 특정 멤버 이름/이메일에만 일치하고 페이지·명령엔 불일치하는 검색어 입력
  Then: "사람 찾기"만 렌더, "페이지"·"명령 실행" 제목 문서에 없음. → [FR-A10]

AC-A15: 세 그룹 모두 무결과
  Given: 팔레트 열림
  When: 페이지·명령·멤버 어느 것도 일치 안 하는 검색어 입력
  Then: "검색 결과가 없습니다" 안내 렌더, 세 그룹 제목 모두 문서에 없음. → [FR-A10]

AC-A16: 그룹 경계 가로지르는 방향키
  Given: SHARED, 검색어 비어 세 그룹 보임, 하이라이트가 "페이지" 마지막 항목
  When: ↓ 키
  Then: 하이라이트가 "사람 찾기" 첫 항목으로 이동. → [FR-A11]

AC-A17: 키보드만으로 명령 실행
  Given: AC-A16 상태에서 하이라이트가 "명령 실행"의 "설정 열기"
  When: Enter
  Then: router.push('/settings') + 닫힘(클릭 없이 키보드만). → [FR-A11]

AC-A18: 사람 찾기 로딩 중 그룹 숨김
  Given: SHARED, 팔레트 열려 멤버 조회 진행 중(미응답)
  When: 조회 진행 시점 확인
  Then: "사람 찾기" 제목 미렌더, 조회 성공 후 그룹 나타남. → [FR-A12]

AC-A19: 사람 조회 401
  Given: SHARED, 팔레트 열려 멤버 조회
  When: 401(ApiError) 실패
  Then: router.push('/login'). → [FR-A12]

AC-A20: 사람 조회 비401 실패
  Given: SHARED, 팔레트 열려 멤버 조회
  When: 500 등 비401 실패
  Then: "사람 찾기" 그룹은 계속 숨김 유지, "페이지"·"명령 실행"은 렌더됨. → [FR-A12]

AC-A21: 트리 로딩 중 전체 로딩 안내
  Given: 사이드바 초기 트리 조회 중(loading=true)
  When: 팔레트를 연다
  Then: 세 그룹 대신 "불러오는 중…" 안내만 렌더. → [FR-A13]

AC-A22: ws 미확정 시 컨텍스트 의존 명령 숨김
  Given: ws가 하나도 없어 선택 ws 미확정(로딩 끝, 대상 없음)
  When: 팔레트를 연다
  Then: "새 페이지 만들기"·"멤버 관리 열기" 미노출, "설정/도움말/테마 전환/로그아웃"은 노출. → [FR-A1]

AC-A23: 재열림 시 검색어 초기화
  Given: 팔레트 열고 "설정" 입력해 "명령 실행"만 남긴 상태
  When: 닫았다가 다시 연다
  Then: 입력창 빈 값, 조건 맞는 세 그룹(SHARED면 사람 포함) 다시 노출. → [FR-A14]

### 기능 B

AC-B1: MembersModal 액션 실패 토스트
  Given: MembersModal 열림, 역할 변경 시도
  When: 비401 실패
  Then: window.alert 미호출, "작업을 처리하지 못했습니다. 잠시 후 다시 시도해 주세요." 토스트(role=alert) 렌더. → [FR-B1]

AC-B2: MembersModal 401 유지
  Given: MembersModal 열림
  When: 변경 액션 401 실패
  Then: router.push('/login'), 토스트 미렌더. → [FR-B1]

AC-B3: MembersModal 다른 액션 동일 경로
  Given: MembersModal 열림
  When: 멤버 내보내기 비401 실패
  Then: "작업을 처리하지 못했습니다. 잠시 후 다시 시도해 주세요." 토스트 렌더. → [FR-B1]

AC-B4: Sidebar 생성 실패 토스트, 트리 유지
  Given: 사이드바 status='ready'
  When: 생성 비401 실패
  Then: 기존 트리 그대로 렌더, "목록을 불러오지 못했습니다" 미렌더, "페이지를 만들지 못했습니다. 잠시 후 다시 시도해 주세요." 토스트 렌더. → [FR-B2/BR-B1]

AC-B5: Sidebar 이름변경 실패 토스트, 트리 유지
  Given: status='ready'
  When: 이름변경 비401 실패
  Then: 트리 유지, "이름을 변경하지 못했습니다. 잠시 후 다시 시도해 주세요." 토스트. → [FR-B2/BR-B1]

AC-B6: Sidebar 아이콘 실패 토스트, 트리 유지
  Given: status='ready'
  When: 아이콘 변경 비401 실패
  Then: 트리 유지, "아이콘을 변경하지 못했습니다. 잠시 후 다시 시도해 주세요." 토스트. → [FR-B2/BR-B1]

AC-B7: Sidebar 아카이브 실패 토스트, 트리 유지
  Given: status='ready', 아카이브 확인 다이얼로그에서 확정
  When: 아카이브 비401 실패
  Then: 트리 유지, "페이지를 아카이브하지 못했습니다. 잠시 후 다시 시도해 주세요." 토스트. → [FR-B2/BR-B1]

AC-B8: Sidebar mutation 401 유지
  Given: status='ready'
  When: 생성/이름변경/아이콘/아카이브 중 하나 401 실패
  Then: router.push('/login'), 토스트도 인라인 오류도 미렌더. → [FR-B2]

AC-B9: 초기 트리 조회 실패는 인라인 유지
  Given: ws 전환 직후 트리 조회 중
  When: 트리 조회 비401 실패
  Then: "목록을 불러오지 못했습니다. 잠시 후 다시 시도해 주세요." 인라인 렌더, 토스트 미렌더. → [FR-B3]

AC-B10: 팔레트 경유 생성 실패 동일 규칙
  Given: ⌘K에서 "새 페이지 만들기" 실행
  When: 생성 비401 실패
  Then: "페이지를 만들지 못했습니다. 잠시 후 다시 시도해 주세요." 토스트 렌더, 사이드바 트리 오류화 안 됨. → [FR-B4]

AC-B11(선택, Could): 재시도 버튼
  Given: 생성 실패 토스트 노출 상태
  When: "다시 시도" 클릭
  Then: 동일 생성 요청 재호출. → [BR-B2]

## 비고 (범위 밖)

- 전용 멤버 프로필 페이지 신설 제외. 사람 찾기는 항상 `/workspace/{wsId}/members`로만 이동.
- 명령 6개 외 추가 명령(워크스페이스 생성, 페이지 삭제/복원 등) 제외.
- 사람 후보에서 본인 제외 로직 제외(본인도 후보 포함).
- 멤버 목록은 팔레트 열릴 때마다 새로 조회(캐시 공유 제외).
- MembersModal 액션 토스트 재시도 버튼 제외(단순 알림 전환만).
- Sidebar mutation 토스트 재시도 버튼(BR-B2/AC-B11)은 Could — 인수 필수 아님.
- "새 워크스페이스" 버튼(no-op) 그대로 둠.
- 명령 항목 아이콘 디자인은 AC 검증 대상 아님(텍스트·동작만).
